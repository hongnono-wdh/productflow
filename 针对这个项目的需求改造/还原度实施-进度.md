# 还原度方案 · 实施进度（loop backlog）

> **执行方式**：loop 自主推进，每轮一个可验证小步——读本清单取下一个未完成任务 → 实现 → 跑测试 → 全绿则本地 commit（不 push）→ 勾掉、更新本清单。
> **停止条件**：全 5 期完成 + 测试全绿 + 已提交 → 结束；连续 2 轮无进展/测试修不回 → 停下报告；遇方案未覆盖、需拍板的设计歧义 → 停下问用户、答后继续。
> **硬纪律**：测试红不提交、不进下一步；**只本地 commit，绝不 push**；保持现有 171+ 测试不回归；只做清单内任务、不擅自扩范围；发现方案需调整先停下提出。
> **依据**：主纲 `还原度提升方案-全流程.md` + 专题 A–E。测试：`python3 -m unittest discover -s productflow/tests -p 'test_*.py'`。版本每期同步 `VERSION`+`SKILL.md`。
