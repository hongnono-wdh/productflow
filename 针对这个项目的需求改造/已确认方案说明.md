## 0. 现状数据模型（复用，不改）

- `pages.json`：`{ pages:[ { id, name, group, status, versions:[{file, platform}], activeVersion, note } ] }` —— **全局页面清单，每页可挂 PC/H5/APP 多平台版本**。
- `canvas.json["4"]`：`{ view, items{ "page:id"→{x,y} }, notes }` —— 画布视图 + 平铺坐标。
- ai-66ef 现状：16 页，每页已能挂三平台版本。

---

## 1. 画布模型 = 模式 × 平台（顺现状）

- **模式**（顶部切换）：设计稿模式 / 流程图模式 —— **同一画板内切换**。
- **平台**（左侧导航）：PC/H5/APP —— **切平台 = 切每页显示的版本**（该平台版本；缺则**占位框**）。**页面清单是全局的、所有平台共享同一批页面**。
- **版本**：每页 `versions` 里按 platform 挑；同平台多候选稿在卡片内切。

| 维度 | 控件     | 设计稿模式                       | 流程图模式                     |
| ---- | -------- | -------------------------------- | ------------------------------ |
| 模式 | 顶部切换 | 页面平铺                         | 页面节点 + 连线                |
| 平台 | 左侧导航 | 每页显示该平台版本（缺 → 占位） | 节点用该平台版本（缺 → 占位） |
| 版本 | 卡片内   | 切同平台候选稿                   | 节点用当前版本                 |

---

## 2. 流程图模式

- **节点 = 全局页面**（缩略图用当前平台版本，缺则占位）；可点高亮 / 跳该页。
- **边 = `{ from页, to页, 操作标注 }`**；一页多出边 / 两页多边；标一个**入口页**。
- **边全局共享一套**（页面全局 → 边平台无关）。
- **生成** = agent 据「全局页面清单 + 每页内容 + 功能模块」自动画**一套**边；用户可加 / 删 / 改标注 / 拖节点 / 重新生成。
- **用途** = 只给人看，不喂 ⑥。

---

## 3. 多平台生图（P4-1 · 顺手修）

- **根因**：现状批量生成只对 `primary` 触发出图（漏 PC/H5），但**数据模型本就支持多平台版本**（`versions[{file,platform}]`）。
- **修法**：批量生成读 `priority`，**对每页 × 每个选定平台触发出图**，填进该页 `versions`。现状 `_auto_page_version(page_id, platform)` 已支持「单页单平台」，**只需批量遍历 priority × pages**。
- **顺序**：按 `priority` 逐平台（APP 全部页 → PC → H5）。
- **数量**：各平台页面数**相同**（全局页面），某平台不需要的页可不出版本 → 占位。

---

## 4. 数据结构（小扩展，向后兼容）

- `pages.json`：**不变**。
- `canvas.json["4"]` 增三个字段：
  - `flow { edges:[{from, to, label}], entry }` —— 流程图边 + 入口页（**全局、平台无关**）；
  - `flowItems { "page:id"→{x,y} }` —— 流程图布局坐标（**与平铺 `items` 分开**两套）；
  - `viewState { mode, platform }` —— 当前模式 + 当前平台。
- 旧项目无这些字段 → 默认「设计稿模式 / 主平台 / 空流程」，兼容。
