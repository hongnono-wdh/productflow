# agent 执行流程规格（AGENT 在 ⑥ 照此执行）

> 本文是 **PRIN-3** 的落地：HUMAN 设计流程，AGENT 按本 runbook 执行。**这是草案，待 HUMAN 审改后 `doc_status: confirmed`。**
> 术语（模块 / 接口 / backend-flow / choice / 全自动模式）见需求文档 §2。

## 0. 输入（前置产物）
AGENT 开始前须已存在：
- `backend-flow.json`（本需求新增产物）——后端流程设计：**模块 → 接口 → 数据流**（节点=模块/接口/数据表，边=calls/reads_from/writes_to）。含**模块顺序**。
- ⑤ 产物：`module-list`（功能模块清单）、`er.md`（数据模型）、API 契约。
- ④ 产物：页面地图（页面 ↔ 模块）、`direction.md`（定稿设计 = 前端实现依据）——因模块 = 全栈单元、含前端页面。
- 向导平台 / ⑤ 选定模板（判断是否涉及后端）。
- ⑤ 已定**数据库**（REQ-9：agent 推荐 + 用户确认；**仅当 DEC-5 判定涉及后端**——无后端无 DB）——决定 schema 方言 / ORM；契约 / schema 锚稳前定好。

## 1. 前置门 · 是否涉及后端（DEC-5）
```
if 项目不涉及后端 (纯静态落地页 T1 / 无后端小工具):
    log("本项目无后端，跳过后端流程开发可视化")   # 不静默——明确提示用户
    结束（不进入下方主循环）
else:
    进入 §1.5
```

## 1.5 系统流程图 · 随时可交互改（非阻塞，无固定闸）
agent 生成 `backend-flow.json` + 系统流程视图后**不设强制放行闸**；HUMAN 可**随时在流程页圈选 / 点节点 + 截图 / 描述怎么改** → agent 改后端设计(`backend-flow.json`)/代码（开发前 / 中 / 后都行；若动接口 → 触发契约变更传播，见需求文档 §7.5）。体现 **PRIN-3**「你随时在图上改设计」。（全自动模式：直接按 agent 设计推进。）

## 2. 主流程 · 全并行 + 每模块自动评审（REQ-7 + DEC-2 + DEC-6 + PRIN-1）
契约就绪后再建地基、**并行 fan-out 全部模块**（无模块间串行门控；模块内前后端也并行；靠契约协调）。数据源 = `backend-flow.json`（**薄关系层**，引用 ⑤ 产物、不重存，#1）：

```
⑴ 【契约就绪 · 软门 · #2 / PRIN-3】亮出流程图 + 各模块接口契约 → 等你 go 或短超时（全自动直接过）→ 契约锚稳（「你设计」的落点；契约稳了再并行，免设计/契约错误导致返工·thrash）
⓪ 先建「地基」: scaffold（用⑤选定框架搭工程）+ 跨模块公共层（DB/ORM · 鉴权 · 公共 schema/组件 · 配置）— 不属任何单模块，先建好
parallel for 模块 M in 全部模块:               # 全并行（像前端多页面并行；靠契约解耦）
    ① 定 M 的接口契约（单一事实来源；⑤ 已有则复用）— PRIN-1：先流程/数据，再接口
    ② 若 M 涉及第三方 key（产品级依赖，DEC-3）: 操作台亮出让用户填/选
    ③ 模块内并行: 前端(按契约,可 mock) ‖ 后端(实现契约 + 数据访问)
    ④ 模块内对接联调: 契约测试 + 集成测试（不一致改实现，不轻易改契约）
    ⑤ 写并跑 M 的测试 → 冒烟
    ⑥ 【模块级自动评审 · DEC-6①】独立 agent 复核(代码审查+测试+契约): 过标「完成」/不过回修(_auto_stage 续跑/熔断)——非阻塞
    ⑦ 更新 M 状态：流程图节点 + **模块进度清单面板**（进行 / 完成 / 需修）
全部模块完成 → §3
```
> 与 ⑥ 现有 ultracode 并行**一致**；加的是「契约优先对接 + 两级自动评审 + 系统流程视图 + 流程页交互改」。HUMAN 无固定闸，随时在流程页圈选/描述改（§1.5）。

## 3. 集成 + 整体流程评审 + 收尾
- 全模块「完成」→ 集成 → 整体冒烟 + **跨模块回归**。
- **整体流程评审（DEC-6 ②）**：独立 agent 审**整个前后端流程**——所有前端页面 / 流程 + 所有后端流程 + 连接(页面↔接口↔数据、完整用户旅程)、闭环一致(无孤儿、契约一致、数据流通)；以**系统流程视图 + 页面地图 + ER** 为面。**无固定 HUMAN 终审闸**——HUMAN 随时在流程页交互改（§1.5）。
- 产出**完整产品**，并在**项目目录**放标准文档集（开发说明 / 使用说明 / **设计方案(从流程图 + ER 导出)** / 接口文档；REQ-8，随产品 git）。
- 系统流程视图：全模块「已完成」，流程↔数据链路完整可下钻（PRIN-2）。

## 4. 全程可视化联动（PRIN-2 / REQ-1·2·4）
每步实时更新系统流程视图：当前模块高亮、完成态、接口↔数据表关联；用户可点节点下钻（模块→接口→单接口处理流程）、跨图跳转。**点前端页面 / 模块 → 高亮其后端接口 + 数据表**（以功能模块为共同锚；可反向：点接口 → 高亮它服务的页面），即 `页面 ↔ 模块 ↔ 接口 ↔ 数据` 一条可跳转链路。

## 5. 复用的现有机制（实现时对接）
- **门控放行** ← `pf_state choice ask/wait` + 全自动模式跳过确认。
- **修复重试 / 熔断** ← `server.py:_auto_stage` 的自动续跑与连续无进展熔断。
- **进度 / 产物 / 模块状态** ← `pf_state`（step/artifact + 新增模块开发状态）。
- **图渲染** ← 系统流程视图（复用 `Canvas.tsx` + 复活的 flow 连线基建）。

## 6. 降级与边界
- 无联网（影响 REQ-6 选型调研）/ 无图渲染能力 → 降级并 `log` 说明，不硬报错（沿用 SKILL.md 降级约定）。
- 两级自动评审是 agent 复核、无需人等；HUMAN 介入 = 随时在流程页圈选 / 描述改（§1.5），非阻塞。
- 本期不含：部署侧 key / 部署流程（DEC-3 scope_now）、函数级拆解（DEC-2）。
