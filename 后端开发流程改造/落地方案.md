## 2. 方案总览（架构取向）

**一句话**：在 ⑤→⑥ 之间加一个**「后端流程」交互画布视图**——agent 生成结构化图产物（真相源），前端复用 ③④ 画布 + **复活 2.22.0 删掉的 flow 连线基建**渲染成可点、可下钻、流程↔数据打通的关联图；仅对**涉及后端**的项目启用。

三根柱子：
1. **数据源（A）**：agent 生成 `backend-flow.json`（结构化图数据，可版本化）——节点 = 模块 / 接口 / 数据表；边 = `calls` / `reads_from` / `writes_to`。两层（模块 → 接口），接口节点可下钻到「单接口处理流程」（mermaid 子图）。
2. **承载 + 交互（B + 交互层）**：新「后端流程」视图，镜像 ④ 的 `mode:'design'|'arch'` 模式机制（`Canvas.tsx:157-159`），复用自控 transform 缩放/平移；**复活** 2.22.0（`68cfe8b`）删掉的贝塞尔连线 / `computeFlowLayout` / 箭头渲染。
3. **打通（PRIN-2 / REQ-4）**：节点可点跳转——接口 → 它 `reads_from/writes_to` 的数据表（连到 ⑤ 的 ER）、页面 → 背后接口；流程与数据流转连成一张可下钻的图。

**条件触发（DEC-5）**：读向导平台 / ⑤ 选定模板（T1 纯静态 = 无后端 → 跳过并提示；T2/T3、带数据层的原生/桌面 = 有后端 → 启用）。
